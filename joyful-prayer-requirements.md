# Joyful 중보기도 기록 앱 요구사항 명세서

> 하나님의 손을 움직이는 중보기도  
> 기쁨의교회 중보기도팀 전용 기도 기록 PWA

---

## 1. 프로젝트 개요

### 1.1 목적
중보기도팀 조원들의 주간 기도 활동(기도 시간, 통독, 암송, 출석 등)을 기록하고, 구글 폼으로 자동 제출하는 모바일 우선 웹앱.

### 1.2 기술 스택
- **프레임워크**: React 18 (Vite 빌드)
- **배포**: Netlify (GitHub 자동 배포)
- **저장소**: localStorage (기도 기록), 외부 fetch (schedule.json)
- **형태**: PWA (Progressive Web App) — 홈화면 설치 가능
- **외부 연동**: 구글 폼 Prefill URL 자동 제출

### 1.3 주차 정의
- **화요일 ~ 월요일** 을 1주로 정의
- `weekKey` = 해당 주 화요일 날짜 (YYYY-MM-DD)
- 제출일 = 다음 화요일

## 2. 공통 요구사항

### 2.1 중보 유형
- **화요중보** / **목회자중보** 두 가지 유형으로 구분
- 최초 설정 및 설정 탭에서 선택 가능
- 유형에 따라 조목록, 구글 폼 URL, 타이틀이 달라짐

### 2.2 데이터 관리
- 모든 기도 기록은 **localStorage**에 저장 (키: `week_YYYY-MM-DD`)
- 앱 캐시 삭제와 무관하게 기록 유지
- **schedule.json** — 조목록, 통독, 암송, 폼 URL을 외부 파일로 관리
  - `public/schedule.json` 위치
  - 앱 빌드 없이 파일만 교체하면 즉시 반영
  - 오프라인 시 localStorage 캐시(`scheduleCache`)로 폴백

### 2.3 주차 탐색
- **이번 주 / 지난 주** 전환 버튼
- 이번 주와 지난 주 두 개만 표시 (그 이전은 통계 탭에서 확인)

### 2.4 UI/UX
- **다크 테마** (배경 `#0D1117`, 골드 강조색 `#C8973A`)
- **한국어** 전용
- **쉬운모드** 토글: html font-size 120% 적용 → 모든 요소 일괄 확대
- **폰트**: Noto Sans KR

## 3. 화면 구성

### 3.1 최초 설정 화면 (SetupScreen)
최초 실행 시에만 표시.

| 항목 | 내용 |
|------|------|
| 중보 유형 선택 | 화요중보 / 목회자중보 버튼 |
| 조 선택 | 유형에 따른 드롭다운 |
| 이름 입력 | 텍스트 입력 |
| 시작하기 버튼 | 모두 입력 시 활성화 |

### 3.2 헤더
| 항목 | 내용 |
|------|------|
| 타이틀 | `⛪ 목회자 중보기도` / `🙏 화요 중보기도` + 연도 |
| 날짜 표시 | `이번 주 YYYY-MM-DD ~ YYYY-MM-DD` (좌측 정렬, 좁은 화면에서 2줄) |
| 제출일 | `제출 YYYY-MM-DD` (골드색) |
| 주간 전환 | 이번 주 / 지난 주 버튼 |
| 이름 표시 | `[02조 홍길동2] 박기성` 형식 (오른쪽 상단) |
| 설정 버튼 | 오른쪽 상단 |

### 3.3 하단 탭 내비게이션
`홈 | 기도 | 통독 | 암송 | 통계`  
탭 아이콘(이모지) + 레이블

## 4. 탭별 기능

### 4.1 홈 탭 (HomeTab)

#### 4.1.1 이번 주 요약 카드 (4개)
- 총 기도시간 → 기도 탭 이동
- 통독 (N/N장) → 통독 탭 이동
- 암송 (완료/미완) → 암송 탭 이동
- 출석 상태 → 통계 탭 이동

#### 4.1.2 기도 파일
- 이번 주 기도 파일로 기도 여부 체크 (완료/미완 토글)

#### 4.1.3 성령의 인도하심
- 텍스트 메모 입력

#### 4.1.4 출석 체크
- 출석 / 지각 / 조퇴 / 결석 버튼 선택
- 지각/조퇴: 시간 및 사유 입력 필드 추가 표시
- 결석: 사유 입력 필드 표시

#### 4.1.5 주간 제출
- 공유 텍스트 미리보기 (접기/펼치기)
- **텍스트 복사** 버튼 (클립보드)
- **공유** 버튼 (네이티브 공유 시트 / 폴백: 복사)
- **제출** 버튼 → 구글 폼 Prefill URL로 새 탭 열기
- 제출 완료 시 날짜 표시

---

### 4.2 기도 탭 (PrayerTab)

#### 4.2.1 요일 선택 바
- 화~월 7개 요일 버튼
- 오늘 강조, 완료(1시간↑) 시 초록색
- 새벽예배/금요예배 아이콘 표시

#### 4.2.2 타이머/스톱워치 카드
- **모드 전환**: 스톱워치 / 타이머
- **원형 SVG 디스플레이**: 진행률 링 + 중앙 시간 표시
- **타이머 설정**: 원 안에서 시/분 드럼롤(스크롤) 선택
  - 위아래 드래그, 마우스 휠, 터치 슬라이드 지원
  - 시간: 0~5시간, 분: 0~55분(5분 단위)
- **타이머 완료 시 알림**:
  - 진동 (Android)
  - 알림음 (Web Audio API, 종소리 3번)
  - 브라우저 알림 (백그라운드 시)
- **시작 / 일시정지 / 저장** 버튼
- **수동 시간 입력**: 드럼롤 UI로 직접 기도 시간 설정 (기록 편집)

#### 4.2.3 기도 시간 보너스
- **새벽예배**: 참석 날 +1시간 자동 반영
- **금요HR예배**: 참석 시 해당 금요일 +2시간 자동 반영
- 새벽예배 요일별 체크박스 (화~월)
- 금요예배 토글 스위치

#### 4.2.4 타이머 백그라운드 동작
- 다른 탭 이동 / 화면 꺼짐 시에도 시간 계속 카운트
- `visibilitychange` + `window.focus` 이벤트로 복귀 시 즉시 보정
- 탭 전환 시 타이머 상태 유지 (App 레벨 state)

---

### 4.3 통독 탭 (ReadingTab)

#### 기능
- 이번 주 통독 범위 표시 (책명 + 장 목록)
- 장별 체크박스
- 전체 완독 체크 (성경 전체 완독)
- 완료 진행률 표시

#### 데이터
- `schedule.json`의 `reading` 배열에서 현재 주차 조회
- 같은 주에 여러 책 가능 (예: 창세기 + 마태복음)

---

### 4.4 암송 탭 (MemoryTab)

#### 기능
- 이번 주 암송 구절 전체 표시 (1절/2절 탭 없음, 한 화면에 모두 표시)
- 암송 녹음 기능 (Web MediaRecorder API)
  - 브라우저별 포맷 자동 감지 (mp4 → webm → ogg)
  - 녹음 완료 시 마이크 자동 해제
  - 재생 배속 선택 (0.5x ~ 2x)
- 암송 완료 체크
- 틀린 글자 수 선택 (0~10)

#### 암송 데이터 구조
- 매주 2절 표시 (이번 주 신규 1절 + 전주 절 반복)
- 실제 신규 구절 수 카운트: `Set`으로 reference 중복 제거

---

### 4.5 통계 탭 (StatsTab)

#### 기간 선택
- **주간** / **월간** / **연간** 탭

#### 주간 목록
- 주차별 카드 (이번 주 강조)
- 총 기도시간, 기도 일수, 새벽기도, 통독, 완독, 제출 여부 표시

#### 월간 목록
- 월별 카드
- 총 기도, 기도 일수, 새벽기도, 통독 장수, 완독, 암송 구절 수(6개 항목)

#### 연간 통계
- 연도 선택 버튼 (올해 🔸 강조)
- 선택한 연도의 상세 통계 카드
- 총 기도시간, 기도 일수, 새벽기도, 통독 장수, 완독, 암송 구절 수
- 주평균 기도시간

#### 암송 카운트 규칙
- 월간/연간: reference 기준 중복 제거 → 실제 신규 구절 수만 집계

---

### 4.6 설정 탭 (SettingsTab)

#### 4.6.1 데이터 백업/복원
- **내보내기**: 모든 기록을 JSON 파일로 다운로드
  - 파일명: `joyful-prayer-backup-YYYY-MM-DD.json`
  - 포함 데이터: `week_*`, `profile`, `easyMode`, `scheduleCache` 등
- **가져오기**: JSON 백업 파일 선택 → 복원 후 앱 새로고침

#### 4.6.2 쉬운모드
- 토글 스위치
- 활성화 시 html font-size 120% 적용
- 설정 localStorage 저장

#### 4.6.3 내 정보
- 중보 유형 선택 (화요중보 / 목회자중보)
- 유형 변경 시 조 목록 초기화
- 조 선택 (드롭다운)
- 이름 입력
- 저장 버튼

#### 4.6.4 관리자 (비밀번호 `1234`)
- 구글 폼 Prefill URL 현황 표시 (목회자중보/화요중보 각각)
- 앱 내장 데이터 현황 (조목록 수, 이번 주 통독/암송 정보)

## 5. 구글 폼 자동 제출

### 5.1 Prefill URL 방식
- `schedule.json`의 `formPrefillUrl`에 유형별 URL 저장
- URL에 포함된 entry 순서 기반으로 매핑 (패턴 인식 아님)

### 5.2 Entry 순서 매핑 (13개 항목)
| 순서 | 항목 | 값 |
|------|------|-----|
| 1 | 날짜 | 제출일 (YYYY-MM-DD) |
| 2 | 조 선택 | 프로필 조 |
| 3 | 이름 | 프로필 이름 |
| 4 | 출석 | 1 (출석/지각/조퇴) / 0 (결석) |
| 5 | 파일기도 | 1 / 0 |
| 6 | 성경통독 | 1 / 0 |
| 7 | 성령인도하심 | 1 / 0 |
| 8 | 성경암송 | 1 (완료·0자) / 0.5 (1~3자) / 0 (미완·4자↑) |
| 9 | 매일기도 | 0~6 (일수) |
| 10 | 총기도시간 | 정수 (시간) |
| 11 | 지각/조퇴 | -0.5 (해당 시) / 미입력 |
| 12 | 사유 | 지각-사유 시간 / 조퇴-사유 시간 / 결석사유 |
| 13 | 전체통독 | `1독 완료` / 미입력 |

### 5.3 제출 검증
- 지각/조퇴 시 시간·사유 미입력이면 경고
- 결석 시 사유 미입력이면 경고
- 제출 후 `submitted: true` 저장 → 버튼 "✓ 재제출"로 변경

## 6. schedule.json 구조

```json
{
  "groupsByType": {
    "목회자중보": ["01조 홍길동1", ...],
    "화요중보":   ["01조 갑돌이1", ...]
  },
  "reading": [
    {
      "startDate": "2026-04-21",
      "endDate":   "2026-04-27",
      "book":      "창세기",
      "chapters":  [1,2,3,4,5]
    }
  ],
  "verses": [
    {
      "startDate": "2026-04-21",
      "endDate":   "2026-04-27",
      "verses": [
        { "reference": "요한복음 15:7", "text": "너희가 내 안에..." }
      ]
    }
  ],
  "formPrefillUrl": {
    "목회자중보": "https://docs.google.com/forms/...",
    "화요중보":   "https://docs.google.com/forms/..."
  }
}
```

## 7. 알림 기능

| 기능 | 조건 | 방법 |
|------|------|------|
| 타이머 완료 진동 | Android | `navigator.vibrate([500,200,500,200,500])` |
| 타이머 완료 알림음 | 모든 브라우저 | Web Audio API, 종소리 3번 |
| 타이머 완료 브라우저 알림 | 백그라운드 + 권한 허용 | `Notification API` |
| 워치 알림 | 페어링 기기 + iPhone 잠금 | 브라우저 알림 자동 전달 |

- 앱 최초 실행 시 알림 권한 요청

## 9. 배포 구조

```
GitHub (kisungdavidpark/joyful-prayer)
    │  push
    ▼
Netlify 자동 빌드 (npm run build)
    │
    ▼
joyfulprayer.netlify.app
```

### 업데이트 시나리오
| 변경 내용 | 방법 |
|-----------|------|
| 조목록/통독/암송/폼URL | `public/schedule.json` 수정 → GitHub 업로드 → 즉시 반영 |
| 앱 기능/UI | `src/App.jsx` 수정 → git push → Netlify 자동 빌드 |
| 기도 기록 | localStorage 저장 → 캐시 삭제해도 유지 |

## 10. 미구현 / 제외된 기능

| 항목 | 이유 |
|------|------|
| 구글 폼 중복 제출 방지 | 폼 자체가 중복 방지 미지원, 재제출 경고로 대체 |
| 애플워치/갤럭시워치 직접 연동 | PWA에서 watchOS/WearOS 앱 연동 불가 (알림 전달만 가능) |
| 구글 스프레드시트 실시간 연동 | CORS 문제로 제거, schedule.json 방식으로 대체 |
| 조목록 앱 내 편집 | schedule.json 수정 후 재배포로 관리 |
